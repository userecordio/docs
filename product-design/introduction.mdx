This is the documentation for our **Product Design System** â€” a single source of truth for the colors, typography, spacing, radius, and core UI components used across our products.

The system is built on **CSS variables (design tokens)** generated from Figma via Kigen, and consumed in code through **Tailwind CSS** and **React \+ TypeScript** components.

## How the system is structured

All tokens are defined as **CSS custom properties** in a single file, `design-tokens.css`. They are layered in three tiers:

<Steps>
  <Step title="Base tokens">
    Raw values â€” the actual pixel, color, or font values. Examples: `--spacing-4: 16px`, `--tailwind-colors-zinc-900: #18181b`, `--radius-rounded-md: 6px`.
  </Step>
  <Step title="Alias tokens">
    Semantic names that point to base tokens, split by theme (`Light` / `Dark`). Examples: `--colors-primary-light: var(--tailwind-colors-zinc-900)`, `--colors-background-dark: var(--tailwind-colors-zinc-950)`.
  </Step>
  <Step title="Theme tokens (--base-*)">
    The tokens components actually consume. They resolve automatically based on the active theme via the `[data-theme]` attribute on the `<html>` or `<body>` element. Examples: `--base-primary`, `--base-background`, `--base-border`.
  </Step>
</Steps>

<Tip>
  When building components, **always use `--base-*` tokens** (e.g. `var(--base-primary)`), never the raw Tailwind color or alias tokens directly. This is what makes light/dark theming automatic.
</Tip>

## Theming

Switch between light and dark mode by setting the `data-theme` attribute:

```html
<!-- Light mode -->
<html data-theme="Light">

<!-- Dark mode -->
<html data-theme="Dark">
```

Every `--base-*` token is redefined inside `[data-theme='Light']` and `[data-theme='Dark']` blocks, so the whole UI re-themes instantly without changing any component code.

## Installation

<Steps>
  <Step title="Add the tokens file">
    Copy `design-tokens.css` into your project (e.g. `src/styles/design-tokens.css`) and import it once at your app root:

    ```tsx
    import "@/styles/design-tokens.css";
    ```
  </Step>
  <Step title="Map tokens into Tailwind">
    In `tailwind.config.ts`, map the semantic tokens so you can use classes like `bg-background`, `text-foreground`, `border-border`, etc.

    ```ts
    // tailwind.config.ts
    export default {
      darkMode: ["selector", '[data-theme="Dark"]'],
      theme: {
        extend: {
          colors: {
            background: "var(--base-background)",
            foreground: "var(--base-foreground)",
            primary: {
              DEFAULT: "var(--base-primary)",
              foreground: "var(--base-primary-foreground)",
            },
            secondary: {
              DEFAULT: "var(--base-secondary)",
              foreground: "var(--base-secondary-foreground)",
            },
            destructive: {
              DEFAULT: "var(--base-destructive)",
              foreground: "var(--base-destructive-foreground)",
            },
            muted: {
              DEFAULT: "var(--base-muted)",
              foreground: "var(--base-muted-foreground)",
            },
            accent: {
              DEFAULT: "var(--base-accent)",
              foreground: "var(--base-accent-foreground)",
            },
            card: {
              DEFAULT: "var(--base-card)",
              foreground: "var(--base-card-foreground)",
            },
            popover: {
              DEFAULT: "var(--base-popover)",
              foreground: "var(--base-popover-foreground)",
            },
            border: "var(--base-border)",
            input: "var(--base-input)",
            ring: "var(--base-ring)",
          },
          borderRadius: {
            sm: "var(--border-radius-sm)",
            DEFAULT: "var(--border-radius-default)",
            md: "var(--border-radius-md)",
            lg: "var(--border-radius-lg)",
            xl: "var(--border-radius-xl)",
            "2xl": "var(--border-radius-2xl)",
            "3xl": "var(--border-radius-3xl)",
            full: "var(--border-radius-full)",
          },
          fontFamily: {
            sans: ["var(--typography-font-family-font-sans)"],
            serif: ["var(--typography-font-family-font-serif)"],
            mono: ["var(--typography-font-family-font-mono)"],
          },
        },
      },
    };
    ```
  </Step>
  <Step title="Add the utility helper">
    Components on the following pages use a small `cn()` helper to merge class names.

    ```ts
    // lib/utils.ts
    import { clsx, type ClassValue } from "clsx";
    import { twMerge } from "tailwind-merge";
    
    export function cn(...inputs: ClassValue[]) {
      return twMerge(clsx(inputs));
    }
    ```

    ```bash
    npm install clsx tailwind-merge class-variance-authority
    ```
  </Step>
  <Step title="Start using components">
    Head to the Design Tokens section for the full token reference and ready-to-use `Button`, `Input`, `Card`, and `Badge` components.
  </Step>
</Steps>

<Note>
  The full raw token file is available at [`design-tokens.css`](/design-tokens/colors) â€” referenced throughout this documentation.
</Note>
